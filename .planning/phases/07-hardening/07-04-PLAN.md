Logging retrofit strategy (from RESEARCH.md):
- JSONFormatter: subclass logging.Formatter, override format() to return json.dumps(dict)
- PipelineLogger: subclass logging.LoggerAdapter, process() merges self.extra into kwargs
- configure_logging(): clears root handlers, adds StreamHandler(JSONFormatter()), suppresses httpx + apscheduler to WARNING
- Each pipeline service: keep module-level logger = logging.getLogger(__name__); add extra={"pipeline_step": "...", "content_history_id": "..."} to key log calls
- Reserve PipelineLogger for daily_pipeline_job orchestration; use extra={} dict at key callsites in services

Anti-pattern to avoid (from RESEARCH.md Pitfall 2):
- Do NOT call logging.basicConfig() after configure_logging() — it will be a no-op but risks double-handler if called first
- Replace the logging.basicConfig(...) block in main.py with configure_logging() — do not add it alongside
</interfaces>

  <action>
    Create src/app/logging_config.py:

    1. Imports: json, logging, datetime, timezone from datetime
    2. JSONFormatter class (subclass of logging.Formatter):
       - format(self, record) -> str:
         - Build dict with: timestamp (record.created as ISO UTC), level (record.levelname), logger (record.name), message (record.getMessage()), pipeline_step (getattr(record, "pipeline_step", "")), content_history_id (getattr(record, "content_history_id", ""))
         - If record.exc_info: add "exc_info" key via self.formatException(record.exc_info)
         - return json.dumps(record_dict, ensure_ascii=False)
    3. PipelineLogger class (subclass of logging.LoggerAdapter):
       - process(self, msg, kwargs) -> tuple:
         - kwargs.setdefault("extra", {}).update(self.extra)
         - return msg, kwargs
       - Usage pattern (document in docstring): plog = PipelineLogger(logger, {"pipeline_step": "script_gen", "content_history_id": ""})
    4. configure_logging(level: int = logging.INFO) -> None:
       - root = logging.getLogger()
       - root.handlers.clear()
       - handler = logging.StreamHandler()
       - handler.setFormatter(JSONFormatter())
       - root.addHandler(handler)
       - root.setLevel(level)
       - logging.getLogger("httpx").setLevel(logging.WARNING)
       - logging.getLogger("apscheduler").setLevel(logging.WARNING)
       - logging.getLogger("telegram").setLevel(logging.WARNING)

    Modify src/app/main.py:
    - Remove the existing logging.basicConfig(...) block (the 3 lines: logging.basicConfig(...), logging.getLogger("httpx")..., logger = logging.getLogger(__name__))
    - Replace with:
      from app.logging_config import configure_logging
      configure_logging()
      logger = logging.getLogger(__name__)
    - Keep all other main.py content unchanged.
