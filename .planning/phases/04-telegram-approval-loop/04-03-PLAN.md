Create `src/app/telegram/handlers/approval_flow.py`. Mirror the structure of mood_flow.py exactly — module docstring, logger, prefix constants, handler functions, register function.

**Prefix constants:**
```python
PREFIX_APPROVE = "appr_approve:"   # 13 chars — callback_data = 13 + 36 UUID = 49 bytes (OK)
PREFIX_REJECT  = "appr_reject:"    # 12 chars — callback_data = 12 + 36 UUID = 48 bytes (OK)
PREFIX_CAUSE   = "appr_cause:"     # 11 chars — callback_data = 11 + 36 UUID + 1 colon + cause_code
                                   # longest: 11+36+1+15 = 63 bytes (technical_error) — just under 64-byte limit
```

**Cause codes and display names:**
```python
CAUSE_OPTIONS = [
    ("Script Error",    "script_error"),
    ("Visual Error",    "visual_error"),
    ("Technical Error", "technical_error"),
    ("Off-topic",       "off_topic"),
]
```

**handle_approve(update, context):**
1. `query = update.callback_query`
2. `await query.answer()` — REQUIRED first (prevents Telegram loading spinner freeze)
3. Extract `content_history_id = query.data[len(PREFIX_APPROVE):]`
4. Instantiate `ApprovalService()` (local import inside function — lazy, avoids circular imports at module load)
5. If `approval_svc.is_already_actioned(content_history_id)`: send "Ya procesado." via `update.effective_chat.send_message()` and return
6. `approval_svc.record_approve(content_history_id)`
7. `approval_svc.clear_constraints_for_approved_run(content_history_id)` — clear active rejection constraints
8. Send follow-up: `await update.effective_chat.send_message("✅ Aprobado — en cola para publicacion")`
9. Log: `logger.info("Approved: content_history_id=%s", content_history_id)`

NOTE: DO NOT call `query.edit_message_reply_markup()` — the original message with its buttons must remain unchanged (user decision: "original message stays unchanged"). Only send new messages via `update.effective_chat.send_message()`.

**handle_reject(update, context):**
1. `query = update.callback_query`
2. `await query.answer()` — REQUIRED
3. Extract `content_history_id = query.data[len(PREFIX_REJECT):]`
4. Build 4-button cause keyboard (each button on its own row — NOT side-by-side):
   ```python
   keyboard = InlineKeyboardMarkup([
       [InlineKeyboardButton(label, callback_data=f"{PREFIX_CAUSE}{content_history_id}:{code}")]
       for label, code in CAUSE_OPTIONS
   ])
   ```
5. Send new message: `await update.effective_chat.send_message("Selecciona la causa del rechazo:", reply_markup=keyboard)`
6. Log: `logger.info("Rejection cause menu sent: content_history_id=%s", content_history_id)`

No idempotency check needed here — showing the cause menu is always safe (the actual rejection is recorded in handle_cause).

**handle_cause(update, context):**
1. `query = update.callback_query`
2. `await query.answer()` — REQUIRED
3. Parse callback_data: `payload = query.data[len(PREFIX_CAUSE):]` then `content_history_id, cause_code = payload.rsplit(":", 1)` — rsplit with maxsplit=1 correctly handles UUIDs (which have hyphens, not colons)
4. Instantiate `ApprovalService()` (local import)
5. If `approval_svc.is_already_actioned(content_history_id)`: send "Ya procesado." and return
6. `approval_svc.record_reject(content_history_id, cause_code)`
7. `approval_svc.write_rejection_constraint(cause_code)`
8. `rejection_count = approval_svc.get_today_rejection_count()`
9. If `rejection_count >= 2`: send "Limite diario alcanzado. El proximo video llega manana a las 07:00 AM." — return without scheduling rerun
10. Otherwise: send `f"⚠️ Rechazado ({cause_code}) — nuevo video en camino en ~15 minutos"` — then call `trigger_immediate_rerun()` (lazy import from `app.scheduler.jobs.daily_pipeline`)
11. Log: `logger.info("Rejected (%s): content_history_id=%s", cause_code, content_history_id)`

**register_approval_handlers(app):**
```python
def register_approval_handlers(app: Application) -> None:
    app.add_handler(CallbackQueryHandler(handle_approve, pattern=f"^{PREFIX_APPROVE}"))
    app.add_handler(CallbackQueryHandler(handle_reject,  pattern=f"^{PREFIX_REJECT}"))
    app.add_handler(CallbackQueryHandler(handle_cause,   pattern=f"^{PREFIX_CAUSE}"))
    logger.info("Approval flow handlers registered.")
```

**Critical implementation notes:**
- All ApprovalService instantiation must be inside the handler body as local import — prevents circular import at module load time (same pattern used in mood_flow.py line 144)
- `update.effective_chat.send_message()` NOT `update.message.reply_text()` — `update.message` is None in CallbackQuery handlers
- All DB reads happen synchronously inside async handlers — Supabase Python client is synchronous; this is acceptable (same pattern as mood_flow handle_duration)
