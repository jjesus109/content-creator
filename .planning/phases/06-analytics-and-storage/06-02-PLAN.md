**Method: `fetch_and_store(content_history_id: str, platform: str, external_post_id: str) -> dict | None`**

This is the main entry point called by harvest_metrics_job. It:
1. Routes to the correct platform fetcher based on `platform` (youtube, instagram, tiktok, facebook)
2. Calls `supabase.table("platform_metrics").insert({...}).execute()` to persist the row
3. Returns the metrics dict on success, None on failure (never raises — fail-soft)

Fields to insert into platform_metrics: `content_history_id`, `platform`, `external_post_id`, `harvested_at` (datetime.now(utc).isoformat()), plus whichever metric fields the platform returns (including `retention_rate` when available).

**Method: `_fetch_youtube(video_id: str) -> dict`**

Uses `requests.get("https://www.googleapis.com/youtube/v3/videos", params={"part": "statistics,contentDetails", "id": video_id, "access_token": access_token}, timeout=15)`.

Get `access_token` by calling `PublishingService()._refresh_youtube_token()` — reuse the existing implementation, do not hand-roll token refresh.

Returns dict with keys: `views`, `likes`, `comments`, `retention_rate`. Map from: `viewCount`, `likeCount`, `commentCount`. For `retention_rate`: YouTube Analytics API returns averageViewPercentage — use `requests.get("https://youtubeanalytics.googleapis.com/v2/reports", params={"ids": "channel==MINE", "metrics": "averageViewPercentage", "dimensions": "video", "filters": f"video=={video_id}", "access_token": access_token})`. If the Analytics call fails, set `retention_rate = None` and log a warning (partial harvest). Cast view/like/comment to int. If `items` list is empty, return `{"views": 0, "likes": 0, "comments": 0, "retention_rate": None}`.

Do NOT attempt to read `dislikeCount` — that field is always 0 since December 2021.

**Method: `_fetch_instagram(media_id: str) -> dict`**

Uses `requests.get(f"https://graph.instagram.com/v18.0/{media_id}/insights", params={"metric": "views,reach,saved,shares", "access_token": settings.instagram_access_token}, timeout=15)`.

IMPORTANT: Use `views` not `video_views` — `video_views` was deprecated in Graph API v21 (January 2025).

For `retention_rate`: Instagram does not expose retention rate directly. Set `retention_rate = None` and store with `harvest_status = 'partial'` indicator (just log a warning — do not add a new column for this).

Returns dict with keys: `views`, `reach`, `saves`, `shares`, `retention_rate`. Parse from `data` list via `{item["name"]: item["values"][0]["value"] for item in response.json().get("data", [])}`. If `data` is empty (account under 1,000 followers threshold or API unavailable), log a warning and return `{"views": None, "reach": None, "saves": None, "shares": None, "retention_rate": None}` — do NOT raise.

**Method: `_fetch_tiktok(video_id: str) -> dict`**

Check `settings.tiktok_access_token` first. If empty string, log `"TikTok access token not configured — skipping harvest"` at WARNING level and return `{"views": None, "likes": None, "shares": None, "retention_rate": None}`.

When token is present: POST to `https://open.tiktokapis.com/v2/video/query/` with `Authorization: Bearer {token}` header, body `{"filters": {"video_ids": [video_id]}, "fields": ["view_count", "like_count", "share_count", "average_time_watched", "total_time_watched"]}`, timeout=15.

Returns dict with keys: `views`, `likes`, `shares`, `retention_rate`. Map from `view_count`, `like_count`, `share_count`. For `retention_rate`: if `average_time_watched` and `total_time_watched` are both present and non-zero, compute `retention_rate = average_time_watched / video_duration * 100` — but TikTok API does not return `video_duration` here, so set `retention_rate = None` and log partial harvest warning.

**Function: `sparkline(values: list[int]) -> str`** (module-level, not a method — pure function, no DB)

```python
SPARK_CHARS = "▁▂▃▄▅▆▇█"

def sparkline(values: list[int]) -> str:
    """Convert a list of up to 4 integers to an 8-level Unicode sparkline string."""
    if not values or max(values) == 0:
        return "▁" * len(values)
    mn, mx = min(values), max(values)
    if mn == mx:
        return "▄" * len(values)
    return "".join(
        SPARK_CHARS[int((v - mn) / (mx - mn) * 7)]
        for v in values
    )
```

**Method: `compute_rolling_average(content_history_id: str, platform: str) -> float`**

Query platform_metrics for the past 28 days:
```python
result = supabase.table("platform_metrics").select("views").eq(
    "content_history_id", content_history_id
).eq("platform", platform).gte(
    "harvested_at", (datetime.now(timezone.utc) - timedelta(days=28)).isoformat()
).execute()
rows = result.data or []
views_list = [r["views"] for r in rows if r.get("views") is not None]
return sum(views_list) / len(views_list) if views_list else 0.0
```

**Method: `check_and_alert_virality(content_history_id: str, platform: str, current_views: int, topic_summary: str, video_date: str) -> bool`**

IMPORTANT: Alert fires every 48h harvest cycle while above threshold — NOT just once. De-duplication check uses time window, not IS NULL.

1. Check de-duplication: query `SELECT virality_alerted_at, harvested_at FROM platform_metrics WHERE content_history_id = $1 AND platform = $2 ORDER BY harvested_at DESC LIMIT 1`. If `virality_alerted_at` is NOT NULL AND `virality_alerted_at > NOW() - 48h`, return False (already alerted within this harvest cycle). If NULL or older than 48h, proceed.
2. Compute `rolling_avg = self.compute_rolling_average(content_history_id, platform)`.
3. Require at least 2 published videos with metrics before alert can fire (baseline requirement from CONTEXT.md): query `SELECT COUNT(DISTINCT content_history_id) FROM platform_metrics WHERE platform = $1`. If count < 2, return False.
4. If `rolling_avg == 0` or `current_views < rolling_avg * 5.0`: return False.
5. Fire alert via `send_alert_sync(format_virality_alert(platform, video_date, current_views))`.
6. Update `virality_alerted_at = datetime.now(utc).isoformat()` on the latest platform_metrics row for this content_history_id + platform.
7. Also mark video as Eternal: update content_history `is_eternal = true, storage_status = 'exempt'` for this content_history_id (viral videos are auto-marked Eternal per CONTEXT.md).
8. Return True.

**Method: `build_weekly_report() -> str`**

Queries for Sunday report. Logic:
1. Get this week's date range: last 7 days. Previous week: 8-14 days ago.
2. For each platform (youtube, instagram, tiktok, facebook), fetch platform_metrics for the current week.
3. Aggregate by content_history_id: for each video, collect per-platform view counts.
4. Top 5 videos by total views this week (sum across all platforms).
5. Top performer of the week = video with highest `retention_rate` in platform_metrics this week (not highest views). Among the top 5, find the one with the max non-null retention_rate. Use `SELECT content_history_id, MAX(retention_rate) as max_retention FROM platform_metrics WHERE harvested_at >= week_start GROUP BY content_history_id ORDER BY max_retention DESC NULLS LAST LIMIT 1`.
6. For each of the 5 videos, compute:
   - `total_views`: sum of views across all platforms this week
   - `pct_change`: (total_views - prev_week_views) / prev_week_views * 100. Set to `None` when `prev_week_views == 0` (first week with no prior data).
   - `platform_breakdown`: dict of {platform: views} for the 4 platforms (None for platforms with no data)
   - `last_4_weeks_views`: list of weekly view totals for the past 4 weeks (for sparkline)
7. Format using `format_weekly_report(top_videos, top_performer_summary)`.

**Function: `format_weekly_report(top_videos: list[dict], top_performer_summary: str = "") -> str`** (module-level)

Shows per-platform breakdown (TikTok, Instagram, Facebook, YouTube) — not aggregate-only. Renders `None` pct_change as "N/A".

```python
def format_weekly_report(top_videos: list[dict], top_performer_summary: str = "") -> str:
    from datetime import date
    lines = [f"REPORTE SEMANAL — {date.today().strftime('%d %b %Y')}\n"]
    if not top_videos:
        lines.append("Sin datos de metricas esta semana.")
        return "\n".join(lines)
    for i, v in enumerate(top_videos[:5], 1):
        spark = sparkline(v.get("last_4_weeks_views", []))
        pct = v.get("pct_change")
        if pct is None:
            pct_str = "N/A"
        elif pct >= 0:
            pct_str = f"+{pct:.0f}%"
        else:
            pct_str = f"{pct:.0f}%"
        # Per-platform breakdown
        breakdown = v.get("platform_breakdown", {})
        yt  = breakdown.get("youtube")
        ig  = breakdown.get("instagram")
        tk  = breakdown.get("tiktok")
        fb  = breakdown.get("facebook")
        def fmt(n): return f"{n:,}" if n is not None else "—"
        platform_line = f"YT: {fmt(yt)} | IG: {fmt(ig)} | TK: {fmt(tk)} | FB: {fmt(fb)}"
        lines.append(
            f"{i}. {v['topic_summary'][:40]}\n"
            f"   {spark} {v['total_views']:,} views ({pct_str})\n"
            f"   {platform_line}"
        )
    if top_performer_summary:
        lines.append(f"\nMejor retencion: {top_performer_summary[:60]}")
    return "\n".join(lines)
```
