This is the main entry point called by harvest_metrics_job. It:
1. Routes to the correct platform fetcher based on `platform`

3. Returns the metrics dict on success, None on failure (never raises — fail-soft)

Fields to insert into platform_metrics: `content_history_id`, `platform`, `external_post_id`, `harvested_at` (datetime.now(utc).isoformat()), plus whichever metric fields the platform returns.

**Method: `_fetch_youtube(video_id: str) -> dict`**

Uses `requests.get("https://www.googleapis.com/youtube/v3/videos", params={"part": "statistics", "id": video_id, "access_token": access_token}, timeout=15)`.

Get `access_token` by calling `PublishingService()._refresh_youtube_token()` — reuse the existing implementation, do not hand-roll token refresh.

Returns dict with keys: `views`, `likes`, `comments`. Map from: `viewCount`, `likeCount`, `commentCount`. Cast all to int. If `items` list is empty, return `{"views": 0, "likes": 0, "comments": 0}`.

Do NOT attempt to read `dislikeCount` — that field is always 0 since December 2021.

**Method: `_fetch_instagram(media_id: str) -> dict`**

Uses `requests.get(f"https://graph.instagram.com/v18.0/{media_id}/insights", params={"metric": "views,reach,saved,shares", "access_token": settings.instagram_access_token}, timeout=15)`.

IMPORTANT: Use `views` not `video_views` — `video_views` was deprecated in Graph API v21 (January 2025).

Returns dict with keys: `views`, `reach`, `saves`, `shares`. Parse from `data` list via `{item["name"]: item["values"][0]["value"] for item in response.json().get("data", [])}`. If `data` is empty (account under 1,000 followers threshold or API unavailable), log a warning and return `{"views": None, "reach": None, "saves": None, "shares": None}` — do NOT raise.

**Method: `_fetch_tiktok(video_id: str) -> dict`**

Check `settings.tiktok_access_token` first. If empty string, log `"TikTok access token not configured — skipping harvest"` at WARNING level and return `{"views": None, "likes": None, "shares": None}`.

When token is present: POST to `https://open.tiktokapis.com/v2/video/query/` with `Authorization: Bearer {token}` header, body `{"filters": {"video_ids": [video_id]}, "fields": ["view_count", "like_count", "share_count"]}`, timeout=15.

Returns dict with keys: `views`, `likes`, `shares`. Map from `view_count`, `like_count`, `share_count`.

**Function: `sparkline(values: list[int]) -> str`** (module-level, not a method — pure function, no DB)

```python
SPARK_CHARS = "▁▂▃▄▅▆▇█"

def sparkline(values: list[int]) -> str:
    """Convert a list of up to 4 integers to an 8-level Unicode sparkline string."""
    if not values or max(values) == 0:
        return "▁" * len(values)
    mn, mx = min(values), max(values)
    if mn == mx:
        return "▄" * len(values)
    return "".join(
        SPARK_CHARS[int((v - mn) / (mx - mn) * 7)]
        for v in values
    )
```

**Method: `compute_rolling_average(content_history_id: str, platform: str) -> float`**

Query platform_metrics for the past 28 days:
```python
result = supabase.table("platform_metrics").select("views").eq(
    "content_history_id", content_history_id
).eq("platform", platform).gte(
    "harvested_at", (datetime.now(timezone.utc) - timedelta(days=28)).isoformat()
).execute()
rows = result.data or []
views_list = [r["views"] for r in rows if r.get("views") is not None]
return sum(views_list) / len(views_list) if views_list else 0.0
```

**Method: `check_and_alert_virality(content_history_id: str, platform: str, current_views: int, topic_summary: str) -> bool`**

1. Check de-duplication: query `SELECT virality_alerted_at FROM platform_metrics WHERE content_history_id = $1 AND platform = $2 ORDER BY harvested_at DESC LIMIT 1`. If `virality_alerted_at` is not NULL, return False (already alerted).
2. Compute `rolling_avg = self.compute_rolling_average(content_history_id, platform)`.
3. If `rolling_avg == 0` or `current_views < rolling_avg * 5.0`: return False.
4. Fire alert via `send_alert_sync(format_virality_alert(...))`.
5. Update `virality_alerted_at = datetime.now(utc).isoformat()` on the latest platform_metrics row.
6. Return True.

**Function: `format_virality_alert(platform, topic_summary, current_views, baseline_avg, pct_above) -> str`** (module-level)

```python
def format_virality_alert(platform: str, topic_summary: str, current_views: int, baseline_avg: float, pct_above: float) -> str:
    return (
        f"ALERTA DE VIRALIDAD\n\n"
        f"Plataforma: {platform.upper()}\n"
        f"Video: {topic_summary[:60]}\n"
        f"Vistas actuales: {current_views:,}\n"
        f"Promedio base (4 sem): {int(baseline_avg):,}\n"
        f"Sobre la media: +{pct_above:.0f}%"
    )
```

**Method: `build_weekly_report() -> str`**

Queries for Sunday report. Logic:
1. Get this week's date range: last 7 days (Mon-Sun). Previous week: 8-14 days ago.
2. Top 5 videos by total views this week:
   ```python
   # For each content_history row with platform_metrics this week,
   # SUM(views) across all platforms, ORDER BY DESC, LIMIT 5
   ```
   Use supabase `.select("content_history_id, views, content_history(topic_summary)")` with join. If Supabase Python client does not support direct joins, do two queries: get top content_history_ids from platform_metrics this week, then fetch topic_summary from content_history.
3. For each of the 5 videos, compute:
   - `total_views`: sum of views across all platforms this week
   - `pct_change`: (total_views - prev_week_views) / prev_week_views * 100 (0.0 if no prior data)
   - `best_platform`: platform with highest views this week
   - `last_4_weeks_views`: list of weekly view totals for the past 4 weeks (for sparkline)
4. Format using `format_weekly_report(top_videos)`.

**Function: `format_weekly_report(top_videos: list[dict]) -> str`** (module-level)

```python
def format_weekly_report(top_videos: list[dict]) -> str:
    from datetime import date
    lines = [f"REPORTE SEMANAL — {date.today().strftime('%d %b %Y')}\n"]
    if not top_videos:
        lines.append("Sin datos de metricas esta semana.")
        return "\n".join(lines)
    for i, v in enumerate(top_videos[:5], 1):
        spark = sparkline(v.get("last_4_weeks_views", []))
        pct = v.get("pct_change", 0.0)
        pct_str = f"+{pct:.0f}%" if pct >= 0 else f"{pct:.0f}%"
        best = v.get("best_platform", "—")
        lines.append(
            f"{i}. {v['topic_summary'][:40]}\n"
            f"   {spark} {v['total_views']:,} views ({pct_str}) | Mejor: {best}"
        )
    return "\n".join(lines)
```
