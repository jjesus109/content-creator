Create the storage lifecycle service (Supabase Storage operations only — no R2) and the Telegram inline handlers for deletion approval, cancellation, and "Save forever".

Purpose: Plan 05's storage_lifecycle_job needs StorageLifecycleService to label warm videos in DB, delete cold files from Supabase Storage, and send 7-day pre-warnings + 45-day deletion confirmations. The Telegram handlers complete the creator-approval loop.

Architecture: Per locked CONTEXT.md decisions:
- Warm (8-45 days) = DB label only. storage_status set to 'warm'. File stays in the same Supabase Storage bucket. NO copy to R2. NO boto3.

- 7-day pre-warning fires at 38 days old, separate from 45-day deletion confirmation.
- "Save forever" button marks video as Eternal and cancels deletion.

**Method: `transition_to_warm(content_history_id: str) -> None`**

Warm transition = DB update ONLY. No file copy, no external storage operation. File stays in the same Supabase Storage bucket.

```python
def transition_to_warm(self, content_history_id: str) -> None:
    """Mark video as warm in DB. File stays in Supabase Storage bucket — no copy needed."""
    supabase = self._get_supabase()
    supabase.table("content_history").update({
        "storage_status": "warm",
        "storage_tier_set_at": datetime.now(timezone.utc).isoformat(),
    }).eq("id", content_history_id).execute()
```

**Method: `delete_from_supabase_storage(content_history_id: str) -> None`**

Cold deletion = delete file from Supabase Storage. DB record is KEPT.

```python
def delete_from_supabase_storage(self, content_history_id: str) -> None:
    """Delete video file from Supabase Storage. DB record (content_history row) is preserved for analytics."""
    supabase = self._get_supabase()
    # Fetch the storage path — typically stored in video_url or a dedicated path column.
    # Check content_history schema: if video_url is a full Supabase Storage URL, extract the path.
    # Pattern: "https://<project>.supabase.co/storage/v1/object/public/videos/<path>"
    row = supabase.table("content_history").select("video_url").eq("id", content_history_id).single().execute().data
    if not row or not row.get("video_url"):
        logger.warning("No video_url found for content_history_id=%s — skipping Supabase deletion", content_history_id[:8])
        return

    video_url = row["video_url"]
    # Extract storage path from URL — everything after "/object/public/videos/"
    import re
    match = re.search(r"/object/public/([^/]+)/(.+)$", video_url)
    if match:
        bucket = match.group(1)
        storage_path = match.group(2)
    else:
        # Fallback: treat video_url as a storage path directly
        bucket = SUPABASE_VIDEO_BUCKET
        storage_path = video_url

    try:
        supabase.storage.from_(bucket).remove([storage_path])
        logger.info("Deleted from Supabase Storage: bucket=%s path=%s", bucket, storage_path)
    except Exception as exc:
        logger.error("Supabase Storage delete failed for %s: %s", content_history_id[:8], exc)
        raise

    # Update DB: mark as deleted. DB record is KEPT (do NOT delete the row).
    supabase.table("content_history").update({
        "storage_status": "deleted",
        "deletion_requested_at": None,
        "storage_tier_set_at": datetime.now(timezone.utc).isoformat(),
    }).eq("id", content_history_id).execute()
```

```python
async def request_deletion_confirmation(self, content_history_id: str, topic_summary: str, days_old: int) -> None:
    """Send 45-day deletion confirmation. Creator must confirm before file is deleted."""
    from telegram import InlineKeyboardButton, InlineKeyboardMarkup
    keyboard = InlineKeyboardMarkup([[
        InlineKeyboardButton("Confirmar Eliminacion", callback_data=f"stor_confirm:{content_history_id}"),
        InlineKeyboardButton("Cancelar",              callback_data=f"stor_cancel:{content_history_id}"),
    ]])
    bot = get_telegram_bot()
    settings = get_settings()
    await bot.send_message(
        chat_id=settings.telegram_creator_id,
        text=(
            f"ALMACENAMIENTO: Video elegible para eliminacion\n\n"
            f"Video: {topic_summary[:60]}\n"
            f"Edad: {days_old} dias\n\n"
            "Si no confirmas en 24 horas, el archivo NO sera eliminado."
        ),
        reply_markup=keyboard,
    )
    supabase = self._get_supabase()
    supabase.table("content_history").update({
        "deletion_requested_at": datetime.now(timezone.utc).isoformat(),
        "storage_status": "pending_deletion",
    }).eq("id", content_history_id).execute()
```

```python
PREFIX_STORAGE_CONFIRM = "stor_confirm:"   # 13 chars + 36-char UUID = 49 bytes (under 64-byte limit)

PREFIX_STORAGE_ETERNAL = "stor_eternal:"   # 13 chars + 36-char UUID = 49 bytes (under 64-byte limit)
```

**`handle_storage_confirm(update, context)`** — async:
1. `await query.answer()` FIRST (prevents Telegram loading spinner freeze — same as all approval handlers).
2. Extract `content_history_id` from `query.data.removeprefix(PREFIX_STORAGE_CONFIRM)`.
3. Check idempotency: fetch `storage_status` from content_history. If already `'deleted'`, reply "Ya eliminado." and return.
4. Check `is_viral` and `is_eternal` — if either is True, reply "Este video esta marcado como exento. No se eliminara." and return. (Safety guard.)
5. Call `StorageLifecycleService().delete_from_supabase_storage(content_history_id)` — this deletes the file from Supabase Storage and updates storage_status to 'deleted' in DB. Note: this may take a moment for large files. The handler is async; await is not needed for the sync method call — it is acceptable to call synchronously since storage deletion is fast.
6. Reply to creator: `"Video eliminado del almacenamiento. El registro de analiticas se conserva."` via `await query.message.reply_text(...)` or `await update.effective_chat.send_message(...)`.

**`handle_storage_cancel(update, context)`** — async:
1. `await query.answer()` FIRST.
2. Extract `content_history_id`.
3. Reset: update `content_history` — `deletion_requested_at = NULL`, `storage_status = 'warm'`.
4. Reply: `"Eliminacion cancelada. El archivo se conserva."`.

**`handle_storage_eternal(update, context)`** — async:
1. `await query.answer()` FIRST.
2. Extract `content_history_id` from `query.data.removeprefix(PREFIX_STORAGE_ETERNAL)`.
3. Check idempotency: if already `is_eternal = true`, reply "Ya marcado como eterno." and return.
4. Update content_history: `is_eternal = true`, `storage_status = 'warm'`, `deletion_requested_at = None`.
5. Reply: `"Video marcado como 'Guardar para siempre'. No sera eliminado automaticamente."`.

**`handle_storage_warn_ok(update, context)`** — async (dismiss the 7-day warning — no action needed beyond acknowledgement):
1. `await query.answer()` FIRST.
2. Reply: `"Entendido. Te avisaremos de nuevo cuando llegue el momento de confirmar la eliminacion."`.
3. No DB update needed — the lifecycle job handles the 45-day transition separately.
