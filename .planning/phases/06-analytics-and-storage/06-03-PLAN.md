**Method: `get_r2_client()`** (internal, returns boto3 S3 client)

```python
import boto3

def get_r2_client(self):
    settings = get_settings()
    return boto3.client(
        service_name="s3",
        endpoint_url=f"https://{settings.r2_account_id}.r2.cloudflarestorage.com",
        aws_access_key_id=settings.r2_access_key_id,
        aws_secret_access_key=settings.r2_secret_access_key,
        region_name="auto",
    )
```

Check `if not settings.r2_account_id` first and raise `RuntimeError("R2 not configured")` so callers know R2 is disabled.

CRITICAL: Never pass `ACL=`, `Tagging=`, or `ChecksumAlgorithm=` to any R2 call — R2 does NOT support these S3 features. Only use `Bucket`, `Key`, and `ExtraArgs={"ContentType": "video/mp4"}`.

**Method: `copy_to_r2(content_history_id: str, video_url: str) -> str`**

1. Build `r2_key = f"videos/{content_history_id}.mp4"`.
2. Download video from Supabase Storage URL using streaming: `response = requests.get(video_url, stream=True, timeout=300)`. Do NOT load all bytes into memory.
3. Upload to R2 using `r2.upload_fileobj(response.raw, Bucket=settings.r2_bucket_name, Key=r2_key, ExtraArgs={"ContentType": "video/mp4"})`.
4. Update content_history: `storage_status = 'warm'`, `r2_key = r2_key`, `storage_tier_set_at = now().isoformat()`.
5. Return `r2_key`.

**Method: `delete_from_r2(content_history_id: str) -> None`**

1. Fetch `r2_key` from `content_history` for this `content_history_id`.
2. If `r2_key` is None, log warning and return (nothing to delete).
3. Call `r2.delete_object(Bucket=settings.r2_bucket_name, Key=r2_key)`.
4. Update content_history: `storage_status = 'deleted'`, `deletion_requested_at = NULL`, `storage_tier_set_at = datetime.now(utc).isoformat()`.

**Method: `request_deletion_confirmation(content_history_id: str, topic_summary: str, days_old: int) -> None`**

Async — sends Telegram inline keyboard confirmation message to creator.

```python
from telegram import InlineKeyboardButton, InlineKeyboardMarkup

keyboard = InlineKeyboardMarkup([[
    InlineKeyboardButton("Confirmar Eliminacion", callback_data=f"stor_confirm:{content_history_id}"),
    InlineKeyboardButton("Cancelar",              callback_data=f"stor_cancel:{content_history_id}"),
]])
bot = get_telegram_bot()
settings = get_settings()
await bot.send_message(
    chat_id=settings.telegram_creator_id,
    text=(
        f"ALMACENAMIENTO: Video elegible para eliminacion\n\n"
        f"Video: {topic_summary[:60]}\n"
        f"Edad: {days_old} dias\n\n"
        "Si no confirmas en 24 horas, el archivo NO sera eliminado."
    ),
    reply_markup=keyboard,
)
```

Then update `content_history`: `deletion_requested_at = datetime.now(utc).isoformat()`, `storage_status = 'pending_deletion'`.

This is an async method. The storage_lifecycle job (Plan 05) will call it via `send_alert_sync` bridge pattern — actually, have the lifecycle job call `asyncio.run(service.request_deletion_confirmation(...))` or use the existing `run_coroutine_threadsafe` pattern from telegram.py. Keep the method async and document this in the docstring.

**Method: `reset_expired_deletion_requests() -> int`**

Called by lifecycle cron when `deletion_requested_at < NOW() - 24h` AND `storage_status = 'pending_deletion'`. Resets these rows to `deletion_requested_at = NULL`, `storage_status = 'warm'`. Returns count of rows reset. This prevents re-prompt flooding (per CONTEXT.md: safe default is NOT deleting).

Follow the exact same structure as `approval_flow.py`. Key differences: prefix names, no cause menu.

```python
PREFIX_STORAGE_CONFIRM = "stor_confirm:"   # 13 chars + 36-char UUID = 49 bytes (under 64-byte limit)
PREFIX_STORAGE_CANCEL  = "stor_cancel:"    # 12 chars + 36-char UUID = 48 bytes (under 64-byte limit)
```

**`handle_storage_confirm(update, context)`** — async:
1. `await query.answer()` FIRST (prevents Telegram loading spinner freeze — same as all approval handlers).
2. Extract `content_history_id` from `query.data.removeprefix(PREFIX_STORAGE_CONFIRM)`.
3. Check idempotency: fetch `storage_status` from content_history. If already `'deleted'`, reply "Ya eliminado." and return.
4. Check `is_viral` and `is_eternal` — if either is True, reply "Este video esta marcado como exento. No se eliminara." and return. (Safety guard in case lifecycle exemption check misfired.)
5. Call `StorageLifecycleService().delete_from_r2(content_history_id)` — synchronous call inside handler (R2 delete is fast, <1s).
6. Reply to creator: `"Video eliminado de almacenamiento R2."` via `update.effective_chat.send_message()`.

**`handle_storage_cancel(update, context)`** — async:
1. `await query.answer()` FIRST.
2. Extract `content_history_id`.
3. Reset: update `content_history` — `deletion_requested_at = NULL`, `storage_status = 'warm'`.
4. Reply: `"Eliminacion cancelada. El archivo se conserva en almacenamiento."`.

**`register_storage_handlers(app: Application) -> None`**:
```python
from telegram.ext import CallbackQueryHandler

```
