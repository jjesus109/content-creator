**Function: `harvest_metrics_job(content_history_id: str, platform: str, external_post_id: str) -> None`**

All three args are strings — required for SQLAlchemyJobStore pickling (same constraint as all Phase 5 jobs).

Logic:
1. Log start: `logger.info("Harvesting %s metrics for content_history_id=%s", platform, content_history_id[:8])`.
2. Call `MetricsService().fetch_and_store(content_history_id, platform, external_post_id)`. Store the returned dict as `metrics`.
3. If `metrics` is None: log warning `"Metrics fetch returned None for %s %s"` and return (fail-soft, do not raise).
4. Extract `current_views = metrics.get("views") or 0`.
5. Fetch `topic_summary` from content_history: `supabase.table("content_history").select("topic_summary").eq("id", content_history_id).single().execute().data.get("topic_summary", "")`.
6. Call `AnalyticsService().check_and_alert_virality(content_history_id, platform, current_views, topic_summary)`.
7. Log completion.

The function must NOT raise under any circumstances — APScheduler logs unhandled exceptions as errors but the job is already complete. Wrap the body in try/except with `logger.error("harvest_metrics_job failed: %s", exc, exc_info=True)`.

**Module-level docstring** explaining: triggered 48h after publish via DateTrigger from platform_publish.py, args must be strings for SQLAlchemy jobstore.

**Imports:**
