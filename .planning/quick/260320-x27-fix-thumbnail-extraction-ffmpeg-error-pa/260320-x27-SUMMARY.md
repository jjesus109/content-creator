# Quick Task 260320-x27: Fix Thumbnail Extraction ffmpeg Error Summary

**One-liner:** Fixed extract_thumbnail() to write MP4 bytes to NamedTemporaryFile before calling ffmpeg, adds -pix_fmt yuvj420p and -strict unofficial to resolve two distinct ffmpeg 7.x failures.

## What Was Done

Two root-cause bugs in `extract_thumbnail()` in `src/app/services/post_copy.py` were fixed:

**Root Cause 1 — Partial-file / EOF on pipe:0 input:**
Kling MP4 files have the `moov` atom at the end of the file. When ffmpeg reads from `pipe:0` (stdin), it cannot seek back to read the moov atom after hitting EOF. The fix writes video bytes to a `NamedTemporaryFile` first, then passes the file path to ffmpeg so it can seek freely.

**Root Cause 2 — ffmpeg 7.x mjpeg encoder YUV range rejection:**
ffmpeg 7.x rejects `yuv420p` (limited range) for the mjpeg encoder without explicit permission. The fix uses `-pix_fmt yuvj420p` (JPEG full-range alias) and adds `-strict unofficial` as belt-and-suspenders.

Additional: `-ss` moved after `-i` for frame-accurate seek when using file input (vs the old fast-seek placement before `-i` which was optimized for pipe input).
